## 1. サービス概要

このサービスは自分が完璧主義だと自覚している人が、例外や体調不良により習慣が途切れ、自己嫌悪するのを防ぐサービスです。  
完璧主義な性格は習慣と相性が良い部分もありますが、行動のハードルが高い、自己嫌悪して再開が遅くなる、などの課題もあります。  
完璧主義な性格の人に合わせた専用の仕組みを使うことで、弱点を抑え、強みを活かしていきます。

---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情
私は習慣を続けることが得意ではありません。  
ある程度続ける事は出来るのですが、ストレスで夜更かししてしまった、やりたいゲームが発売され没頭してしまった、などの普段のリズムが崩れた際によく習慣が途切れてしまいます。  
そのたびに「また同じことを繰り返している」「何も成長していない」と自己嫌悪になりました。

習慣を続けるために心理学や習慣に関する本を買って読んだり、youtubeなどを参考にNotionで習慣管理をしていたこともあります。習慣に関する知識は人一倍あると自負しています。  
ですが、未だに習慣を続ける事に苦手意識があります。

---

### 2-2. なぜそれが「気になった」のか

今になって冷静に自分を振り返ってみると「完璧主義な性格」が一番良くなかったのだと理解できました。  
習慣を続けようとこだわり過ぎるあまり複雑になり、続けるハードルが上がってしまいました。イレギュラーな状況に対応できなくなります。

スモールステップで始める、考えすぎずとにかく着手する、といった完璧主義の対策をしていたのですが、
これらだけでは足りませんでした。対策したつもりになっていました。  
完璧主義な性格に合わせた専用の仕組みが必要なのだと気づきました。

---

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと

続けていた習慣がイレギュラーな要因で途切れてしまった。  
そのせいで自己嫌悪になり、習慣の再開に時間がかかってしまう。

---

### 3-2. 本当に解決したい課題は何か

「完璧主義な考え方による習慣が途切れるのを防ぎたい」

完璧主義な人が習慣を考える際、100点を目指すあまり高い目標を掲げてしまいがちです。  
ですが人間なのでその日の感情にブレがあり、最初に設定した目標が達成できない場合もあります。  
その結果自己嫌悪になってしまいます。
対策として、達成しやすい最低限の目標が必要です。

しかも限界までやることを詰め込むのでバッファがなく、イレギュラーな要因に弱くなります。
例外を考慮して余裕を作るか、例外に対応したルールが必要です。

加えて、完璧主義という自分の性格をしっかりと理解する必要があります。  
完璧主義な性格は習慣と相性が良い部分もあり、性格自体が悪いのではありません。
自己嫌悪にならないよう失敗するパターンを認識し、仕組みによって完璧主義を制御する必要があります。

---

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー

「自分は完璧主義な性格だと自覚していて、習慣の継続が上手くいかない人」

習慣を継続したい、生活を改善したい意思がある。  
性格の影響で行動するスピードが遅くなる、達成までのハードルが高くなってしまう。  
そのせいで、メンタルの乱れやイレギュラーな要因に弱く、習慣が途切れてしまう。  
習慣が途切れると自己嫌悪してしまい、再開にも時間がかかってしまう。  
そして完璧主義なのは分かっているけど、性格はどうしようも無いと考えている。

このようなユーザーを想定しています。
まずは、完璧主義という性格を正しく理解し、自分を否定しないようにすることが大切です。  
そのうえで行動のハードルを下げ、イレギュラーな状況に対応できるようにしていきます。

---

### 4-2. 自分とユーザーの距離

ユーザーとの距離感で最も近いのは自分自身です。今までの経験から、まず自分自身がこのサービスが欲しいと思いました。  
まずは自分が満足できないと他の人にも満足してもらえないと思います。

### 4-3. 実際に使われる可能性について(需要・存在確認)

まず最初に自分で使ってみます。あとは家族や同期の人に使ってもらえると思います。

完璧主義な人はその性格から、自分の行動を管理したい・コントロールしたいと考える人は多いと思います。
ですがその性格のせいで上手くいかない人も多いので需要はあると考えています。

習慣の管理はNotionでも出来ますが、Notionは高い拡張性があります。  
完璧主義な人にとってはそのせいでこだわり過ぎてしまい、実行までのスピードが遅くなったり、
実行のハードルが高くなってしまいます。私もこの経験があります。  
完璧主義な人にはその性格に合わせた専用の仕組みを持ったサービスが必要です。

## 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)

サービスを導入したらまず最初に、完璧主義のリフレーミングを行います。
性格について解説し正しく理解してもらいます。

習慣を登録します。このとき、習慣内容と最低ラインを必須で入力します。  
習慣のルールを登録し、体調不良などの場合にどうするか予め決めておきます。  
例：体調不良の場合は代わりに他の習慣をすればOKにする。など

一日の夜に習慣をチェックし四段階で評価します。例外が起きた際はその内容と条件を記録します。  
週に一回振り返りをします。例外が起きる理由を把握し、習慣内容やルールを調整します。

習慣を身につけるため毎日継続します。習慣の定着に必要とされる66日が目安になります。  
これを達成できればサービスが無くても習慣が継続出来ますし、そのまま記録を続けても大丈夫です。  
また、完璧主義のリフレーミングは定期的にリマインドし、ユーザーに正しい認識を続けてもらいます。

### 5-2. それでも自分が作りたい理由

習慣を続けるのは大変です。これは誰もが思っていることだと思いますが、完璧主義な性格を持つ人はさらに大変です。挫折するともう嫌になり、最悪二度とやらなくなってしまいます。

私は本やネットから習慣に関する知識を集め、Notionで習慣を管理していました。
Notionのテンプレートを元に自分で使いやすく改良する、PCとスマホ両方で使いやすくする、などこだわり過ぎていました。  
しかもそれが良いことだと思いこんでしまい、行動のハードルが上がってしまいました。

自分は完璧主義だから、完了主義になろう、直ぐに着手しよう、と意識していましたが、
時間が経つと意識が薄れてしまい、「これじゃ足りない、もっとやらなきゃ」と無意識にやることを増やしてしまっていました。振り返りやリマインドが足りてなかったです。

この過去の経験から完璧主義な人が習慣を継続できるよう、もっとシンプルで拡張性はほどほどに、性格に合ったサービスを作りたいと思いました。

### 5-3. 差別化を一文で決める

このアプリは、完璧主義な人が習慣を継続するのを助ける専用アプリです。

## 7. このアプリで実現すること

### MVPで作る機能
- ユーザー登録・ログイン機能（メールアドレス＋パスワード）  
- 習慣の登録・編集機能、一覧表示、詳細表示、削除  
習慣を登録する際は、習慣内容と最低ラインの二つが必須で、同じ内容は登録できません。
最低ラインを設定することでハードルを下げ、自己嫌悪を防ぎます。

- 習慣ルール設定  
習慣ごとにルールを設定し、例外が起こった場合や体調不要の場合の対応を設定します。
想定外の事態を事前に想定することで、挫折を防ぎます。
例：急用が入った場合は、最低ラインを実施する。など

- 習慣チェック  
一日の夜に「全ての習慣を達成」「最低ラインを達成」「例外で未達成」「伸びしろあり」の四つの評価から選びます。
例外が起きた際は、その内容と条件を記録します。

- 習慣ポイント記録  
習慣ごとにポイントを加算し、継続状況を可視化します。  
全ての習慣・最低ラインを達成すると1pt、それ以外は0ptですが、再開すると2pt加算するようにします。減点は無しです。習慣が○××○の場合は、+1, 0, 0, +2になります。  
これにより再開を促し継続しやすくなるとともに、自己嫌悪を防ぎます。  
ユーザーは全ての習慣ポイントを合計した「経験値」を持ちます。ポイントが積み上がることで自信に繋がります。

- 振り返り機能  
週に一回振り返りをします。例外が起きる条件を把握し、習慣内容やルールを調整します。  
以下の様な改善点を提案し、完璧主義の欠点を補います。  
・一日の予定に余裕はありますか？  
・もっと簡略化して最低限にできそうですか？一分でも大丈夫！  
・例外が起きた時はどうしますか？  

- 完璧主義のリフレーミング  
完璧主義な性格が悪いものではないとユーザーに解説し、定期的にリマインドします。

これらの機能により、ユーザーは完璧主義を正しく理解し、行動のハードルを下げ、自己嫌悪を防ぐことで、継続と再開がしやすくなります。  
MVPリリースでは最低限の機能に抑え、本リリースの機能でこれらの要素をもっと強化していきます。


### 本リリースで作る機能

- 再開サポート  
生きていればやむを得ず、習慣が途切れることもあると思います。
そんな時に再開しやすいよう、アプリを起動していない時間が一定期間になると再開を促すメッセージを送信します。
- 習慣通知機能
- 振り返り機能の改善提案パターンの追加
- 振り返りの曜日指定（デフォルトは日曜日）
- 継続状況をカレンダーで表示（再開した日を強調する）
- 習慣ステータスの設定  
継続中、休止中、再開予定など
- 目標設定  
一週間、一ヶ月、66日など段階的に設定
- 再開ラインの設定  
○日以内に再開すればその分の習慣ポイントが貰える。
- レベルアップ機能  
経験値が一定に達するとレベルアップする。称号が変化する。  
レベルに応じて機能解放するのもありかも。要検討
- 習慣テンプレートの提供機能
- 設定機能  
ユーザーに合わせて、期間や機能のONOFFなど細かいところを設定できるようにします。

## 8. このアプリの懸念点とその対策

専用化するために機能を増やしすぎるとアプリ自体が複雑になってしまいます。  
これにより起動するのが面倒になったら本末転倒です。機能はシンプルに、ユーザーが簡単に継続しやすいUIUXを意識して開発する必要があります。

## 10. 技術スタック（手段としての技術）

### 10-1. 使用予定の技術

- フレームワーク：Ruby on Rails7.2, Hotwire（Turbo）, Tailwind CSS + daisyUI
- DB：PostgreSQL
- デプロイ先：Render
- 使用予定ライブラリ：devise, acts-as-taggable-on, pagy

---

### 10-2. 技術選定の理由

アプリ開発初心者でも使いやすく、ユーザが使いやすい実装ができる技術を選びました。

フレームワークは情報が豊富なRails7.2、Rails標準のHotwire（Turbo）、そして簡単に綺麗な画面が作れるTailwind CSS + daisyUIにしました。

DBはRailsで一般的に使われているPostgreSQLにしました。これならRenderでも使用できます。

デプロイはGitHub連携で初心者でもやりやすく、無料で使用できるRenderにしました。

ライブラリはユーザー認証、タグ機能は一般的なものを、ページネーションは動作が早いpagyにしました。

技術選定をするのが初めてなので、デザインを変更したい場合など修正箇所が多くならないか不安はあります。
