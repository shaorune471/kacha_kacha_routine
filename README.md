# カチャカチャルーティーン⚙️

行動という歯車を噛み合わせ、生活を自動化しよう！

## 1. サービス概要（※最後に書いてOK）

状況に合わせたルーティーンを複数作成し、どんな状況でも自分のやるべきことが直ぐに分かる、習慣の管理と継続するためのサービスです。

事前にルーティーンを作成することで、忙しい状況や余裕のない状況でも、夜更かしなどの悪い習慣による悪循環を無くすことが出来ます。  
これにより体調不良などを防ぐことが出来ます。

---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

学習期間中に生活習慣がぐちゃぐちゃになってしまい、体調が悪くなりました。  
学習のやる気が下がり思うように進まず、焦りや不安、自己嫌悪を感じるようになりました。

しかも生活習慣は一度崩れると元に戻すのが大変で、悪循環を断ち切ることがなかなかできませんでした。

この体験から、いつ、何をするのか、どんな効果やメリットがあるのかなどを一覧で見れると便利だと思いました。

---

### 2-2. なぜそれが「気になった」のか

日々の習慣の力がとても大きいものだと考えているから。
正しい習慣を継続することで、体内時計や自律神経が整ったり、メンタルが安定するなど、様々なメリットがあるから。

今までに何度か自分に合った習慣を考えたが、継続できなかったり、継続できても仕事が忙しくて途切れたりすることがよくありました。

夜更かしなどのたった一つの悪い習慣が原因で、悪循環に陥ることがよくありました。  
これは意志が弱いわけではなく、悪循環にならない仕組みを作る必要があります。その仕組みを作りやすいサービスが欲しいと思いました。

---

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと

生活習慣が崩れたことにより体調が悪くなり、学習のやる気も下がってしまった。

---

### 3-2. 本当に解決したい課題は何か

生活習慣の乱れによる体調不良を防ぎたい。生活習慣が乱れると体内時計や自律神経が乱れてしまうから。

良い習慣を続けるためには、新しい行動を今ある習慣に紐づけることが重要です。  
自分の行動を自動化することで、感情に頼らずに継続することができると考えています。

行動という歯車を噛み合わせて、途切れることなく自動的に動いていくのが理想です。

---

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー

- 現状に余裕が無くて生活習慣が乱れてしまう人  
忙しくて習慣が途切れても、直ぐ確認して再開することができる。通常時と繁忙期など複数のルーティーンを準備できる。

- 体調不良を改善したい人  
メンタル改善や自律神経を整えるなどのルーティーンのテンプレートを用意して、これを元に自分用に修正できる。  
同じような状況の人が共有したルーティーンを参考にして、自分に取り込むことができる。  
行動にどんなメリットがあるか意識することで継続しやすくなる。何も意識しないとただの作業になってしまう。


- 自分の習慣や行動をまとめて管理したい人  
習慣のリストアップや編集、行動にどんな効果があるのか一目で把握することができる。

---

### 4-2. 自分とユーザーの距離

ユーザーとの距離感で最も近いのは自分自身です。今までの経験から、まず自分自身がこのサービスが欲しいと思いました。  
まずは自分が満足できないと他の人にも満足してもらえないと思います。

### 4-3. 実際に使われる可能性について(需要・存在確認)

まず最初に自分で使ってみます。あとは家族や同期の人に使ってもらえると思います。
習慣の管理や継続は誰にでもできるし、誰もがやると効果のあるものだと思うので、需要はあると考えています。

習慣の管理はNotionやテキストエディタでも出来ますが、使うまでの準備や管理が大変です。  
このサービスを使うことで、自分に合った習慣を考え作成し、管理することが簡単にできるようになります。

## 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)

体調を崩してしまう人や夜に眠れない人、習慣を改善したいと思う人達の助けになります。  
また、習慣は一度考えただけで終わらず毎日継続するものなので、このアプリも継続して利用してもらえると考えています。

また、習慣のテンプレートを用意したり、同じような境遇の人が共有したルーティーンを参考にして、自分に合った習慣を考えるハードルを下げたいと考えています。

### 5-2. それでも自分が作りたい理由

一つの習慣を続けることも大切ですが、今の自分がやりたい事は、
- 悪い習慣による悪循環を事前に防ぎたい。断ち切れるようにしたい。
- 迷いを無くしたい。状況に合わせてまとまった習慣を自動で行動できるようにしたい。
- やるべき習慣を忘れてしまっても、直ぐに確認して再開できるようにしたい。

この３つです。これらを実現するために、

- 状況に合わせたルーティーンを事前に用意できる（悪循環に入る前/入った後）
- ルーティーンを自分に合わせて改良しやすい

これらの要素を持ったサービスを開発したいと考えました。

まずは習慣の管理や行動の整理ができるようにし、その後の展開で一つの習慣にフォーカスして継続しやすくする機能も実装したいと考えています。

また、機能を増やしすぎず、シンプルで継続しやすいことを常に意識して開発していきたいです。

### 5-3. 差別化を一文で決める

状況に合わせて複数のルーティーンを作成し行動を管理することで、忙しい人の悪い習慣による悪循環を防ぐことが出来ます。

## 6. このサービスで提供したい価値

### 6-1. ユーザーの変化

このサービスを使うことでユーザーは習慣を考えやすくなり、余裕がない状況でも自分のやるべき行動が直ぐに確認できます。

この行動リストを順番に進めることで、良い習慣を作り悪循環を無くす事ができます。これにより体調不良などを防ぐことが出来ます。

また、今まで習慣を意識したことが無い人でも、自分の習慣を考えるきっかけになるようなサービスにしたいです。

### 6-2. 価値を一文で表す

忙しくても体調が悪くても、悪循環にならないための自分の道しるべになるサービスです。

## 7. このアプリで実現すること

### MVPで作る機能
- ユーザー登録・ログイン機能（メールアドレス＋パスワード）
- ルーティーン作成・編集機能、一覧表示、詳細表示、削除  
ルーティーンは行動タイトル、行動時間（任意）、開始時間（任意）、タグ（任意）の一覧です。  
ユーザーは状況に合わせた複数のルーティーンを作成することが出来ます。
（平日用、休日用、繁忙期、睡眠不足の改善など）  
ルーティーンを事前に用意することで、どんな状況でも自分のやるべき行動がスマホで直ぐに確認でき、迷いが無くなります。  
また、生活に余裕がなくやるべき行動を忘れてしまっても、直ぐ確認して再開することが出来ます。

- 行動の作成・編集機能、一覧表示、詳細表示、削除  
行動はタイトル、行動時間（任意）、タグ（任意）、メモ（任意）を持ちます。  
メモにはその行動の詳細を書いたり、メリットを書いたりして意識しやすくすることが出来ます。行動のメリットを意識することで作業感が軽減され、習慣を継続しやすくなります。  
また、行動をしないデメリットを書くのも効果的です。アプリの説明で選択肢を提示し、ユーザーが自分に合わせて内容を考えることが出来ます。
作成した行動は、ルーティーン作成・編集の際に呼び出すことが出来ます。ルーティーン作成の負担が軽減されることで、アプリを使うハードルを下げ、習慣が継続しやすくなります。

- 行動のタグ機能  
行動にタグを設定することで、目的に合った行動を探しやすくなります。これにより、ユーザーの負担を軽減することができます。  
ルーティーンの詳細表示でタグを表示することで、その行動にどんな意味があるかを意識できるようになります。行動の意味を意識することで、習慣を継続しやすくなります。
（メンタル改善、睡眠改善、美容、アンチエイジングなど）

### 本リリースで作る機能
- パスワードリセット
- ルーティーン・行動のテンプレート提供機能（管理画面追加）  
ユーザーによっては一からルーティーンを考えることは大変ですが、万人に効果のあるルーティーンや行動を提供することで、考えるハードルが下がり、自分に合ったルーティーンを作成しやすくなります。

- SNS投稿機能(X)  
他人に自分のルーティーンを共有したり、継続する宣言をすることが出来ます。  
他人に宣言することで、習慣の継続率を上げる効果もあります。

- プロフィール機能、フォロー機能  
ルーティーンを公開した際やグループ機能で使用するプロフィール情報を設定します。これらの機能を利用しない場合は、設定する必要はありません。

- ルーティーン・行動の公開とコメント機能  
ルーティーンや行動を公開することで、他人からコメントでアドバイスや共感など、リアクションを受け取ることが出来ます。  
これにより、ルーティーンの改善やモチベーションの向上が出来ます。  
自分と同じ状況の人が公開したルーティーンを参考にして、自分に取り込むことが出来ます。ユーザーは自分に合ったルーティーンを考えやすくなります。

- グループ作成機能（限定公開、チャット）  
特定のユーザーとグループを作成し、ルーティーンを限定公開することが出来ます。  
グループチャットでコミュニケーションを取ることで、習慣の継続率を上げることが出来ます。

- UIUXの調整（ユーザーの負担を軽減する）

## 10. 技術スタック（手段としての技術）

### 10-1. 使用予定の技術

- フレームワーク：Ruby on Rails7.2, Hotwire（Turbo）, Tailwind CSS + daisyUI
- DB：PostgreSQL
- デプロイ先：Render
- 使用予定ライブラリ：devise, acts-as-taggable-on, pagy

---

### 10-2. 技術選定の理由

アプリ開発初心者でも使いやすく、ユーザが使いやすい実装ができる技術を選びました。

フレームワークは情報が豊富なRails7.2、Rails標準のHotwire（Turbo）、そして簡単に綺麗な画面が作れるTailwind CSS + daisyUIにしました。

DBはRailsで一般的に使われているPostgreSQLにしました。これならRenderでも使用できます。

デプロイはGitHub連携で初心者でもやりやすく、無料で使用できるRenderにしました。

ライブラリはユーザー認証、タグ機能は一般的なものを、ページネーションは動作が早いpagyにしました。

技術選定をするのが初めてなので、デザインを変更したい場合など修正箇所が多くならないか不安はあります。
